    ECMAScript 'throw' expressions

    Introduction

    This proposal defines new syntax to throw exceptions from within an expression context.

    See the proposal repository for background material and discussion.

    1ECMAScript Language: Expressions

    1.1Unary Operators

    Syntax

    UnaryExpression[Yield, Await]:UpdateExpression[?Yield, ?Await] deleteUnaryExpression[?Yield, ?Await] voidUnaryExpression[?Yield, ?Await] typeofUnaryExpression[?Yield, ?Await] throwUnaryExpression[?Yield, ?Await] +UnaryExpression[?Yield, ?Await] -UnaryExpression[?Yield, ?Await] ~UnaryExpression[?Yield, ?Await] !UnaryExpression[?Yield, ?Await] [+Await]AwaitExpression[?Yield]

    1.1.1Static Semantics: IsFunctionDefinition

    UnaryExpression:deleteUnaryExpression voidUnaryExpression typeofUnaryExpression throwUnaryExpression[?Yield, ?Await] +UnaryExpression -UnaryExpression ~UnaryExpression !UnaryExpression AwaitExpression

    1.3The throw Operator

    1.3.1Runtime Semantics: Evaluation

    UnaryExpression:throwUnaryExpression
    1. Let exprRef be the result of evaluating UnaryExpression.
    2. Let exprValue be ? GetValue(exprRef).
    3. Return Completion{[[Type]]: throw, [[Value]]: exprValue, [[Target]]: empty}.
